## 1. 写在前面

### 1.1 这个项目是干什么的

`mimo2api` 是一个 API 中转网关，把小米 MIMO 模型转换成 OpenAI / Anthropic 兼容接口，方便你用各种支持 OpenAI 的客户端（如 ChatGPT Next Web、Cherry Studio 等）调用 MIMO 模型。

它由两个部分组成：

- **网关（main.py）**：跑在你的服务器上，对外提供 `/v1/chat/completions` 这种标准 API 接口。
- **节点（bridge.py）**：网关启动后会自动管理小米账号，让账号通过 WebSocket 主动连接到网关，承担实际的请求转发。

### 1.2 你需要准备什么

- 一台能联网的电脑或服务器（macOS / Linux）
- 至少一个能登录小米 AI Studio 的账号凭证（不提供账号本身，需要你自己有）
- 大约 30 分钟时间

### 1.3 文档约定

- 文档里的命令请**逐行复制**到终端执行，不要一次性贴一大段。
- 命令前面的 `$` 不要复制，那只是提示符。
- 看到 `# 注释` 是说明，不用执行。
- 出现错误别慌，文末"常见问题"章节大概率有答案。

## 2. 第一步：检查你的电脑环境

### 2.1 打开终端

- **macOS**：按 `Command + 空格` 打开 Spotlight，输入 `Terminal` 回车。
- **Linux**：你应该已经在 SSH 终端里了。

### 2.2 确认操作系统

执行：

```bash
uname -a
```

如果输出包含 `Darwin` 就是 macOS，包含 `Linux` 就是 Linux。

### 2.3 检查是否已安装 Python

```bash
python3 --version
```

可能出现三种结果：

**结果 A**：输出 `Python 3.10.x` 或更高（比如 3.11、3.12）。恭喜，跳到第三步。

**结果 B**：输出 `Python 3.9.x` 或更低。版本太老，需要升级。看第二步。

**结果 C**：提示 `command not found`。没装 Python，看第二步。

## 3. 第二步：安装 Python 3.10 或更高版本

### 3.1 macOS 上安装

最简单的方法是用 Homebrew。先检查是否有 Homebrew：

```bash
brew --version
```

如果提示 `command not found`，先装 Homebrew：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

装完 Homebrew，再装 Python：

```bash
brew install python@3.11
```

装完后验证：

```bash
python3 --version
```

### 3.2 Ubuntu / Debian Linux 上安装

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip
python3 --version
```

如果系统自带的 Python 版本低于 3.10，需要装新版：

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update
sudo apt install -y python3.11 python3.11-venv python3.11-pip
python3.11 --version
```

后续命令里看到 `python3` 都替换成 `python3.11`。

### 3.3 CentOS / RHEL Linux 上安装

```bash
sudo yum install -y python3 python3-pip
python3 --version
```

如果版本太低，参考 [Python 官方文档](https://www.python.org/downloads/source/) 编译安装新版本，或者考虑换 Ubuntu。

## 5. 第四步：创建 Python 虚拟环境

### 5.1 为什么要用虚拟环境

把这个项目的依赖隔离开来，不和系统里其他 Python 项目冲突。你可以理解成给这个项目单独建一个干净的"小房间"。

### 5.2 创建虚拟环境

确保你在项目目录里（`mimi3-master`），然后执行：

```bash
python3 -m venv .venv
```

执行后会生成一个 `.venv` 文件夹（默认隐藏，用 `ls -la` 才能看到）。这一步可能需要 5 到 30 秒，请耐心等待。

### 5.3 验证创建成功

```bash
ls -la .venv
```

应该能看到 `bin`、`lib`、`pyvenv.cfg` 等内容，说明创建成功。

### 8.1 复制环境变量模板

```bash
cp env.example .env
```

### 8.2 编辑 .env 文件

用任何文本编辑器打开 `.env`。如果用命令行：

```bash
nano .env
```

或者用图形界面（macOS）：

```bash

```

### 8.3 配置内容讲解

文件里所有配置项都被注释掉了（前面有 `#`）。要启用某项，把行首的 `#` 删掉再改值。

**最简配置（本地测试用）**：

```
SERVER_HOST=0.0.0.0
SERVER_PORT=8000
```

这两行让服务监听所有网络接口的 8000 端口。

**推荐配置（个人使用）**：

```
SERVER_HOST=0.0.0.0
SERVER_PORT=8000

# 客户端调用 /v1/* 必须带的 API Key，类似 OpenAI 的 sk-xxx
MIMO_RELAY_OPENAI_KEY=sk-换成你自己的一长串随机字符

# WebUI 控制面板登录账号密码
MIMO_WEBUI_USERNAME=admin
MIMO_WEBUI_PASSWORD=换成你自己的密码

# WebUI 会话签名密钥（随便写一长串随机字符即可）
MIMO_WEBUI_SECRET=换成至少32位的随机字符串
```

### 8.4 生成随机密钥的小技巧

打开另一个终端窗口执行：

```bash
openssl rand -hex 32
```

会输出一串 64 位的随机字符，直接复制粘贴当 `MIMO_WEBUI_SECRET` 用。

### 8.5 保存退出

- nano 编辑器：按 `Ctrl + O` 保存，回车确认，再按 `Ctrl + X` 退出。
- macOS TextEdit：按 `Command + S` 保存，关闭窗口。

## 9. 第八步：准备用户凭证（重要）

### 9.1 凭证从哪里来

凭证就是登录小米 AI Studio 网站后浏览器里的 Cookie。项目本身**不提供账号**，需要你自己有能登录 https://aistudio.xiaomimimo.com 的小米账号。

### 9.2 获取凭证的步骤

1. 用 Chrome / Edge 浏览器打开 https://aistudio.xiaomimimo.com 并登录。
2. 按 `F12` 打开开发者工具，切到 **Network（网络）** 标签。
3. 刷新一下页面，在请求列表里随便点一个发到 `xiaomimimo.com` 的请求。
4. 在右侧找到 **Request Headers（请求头）**，往下翻找到 `Cookie:` 那一行。
5. 复制 `Cookie:` 后面那一整串内容（很长，包含好几对 `key=value;`）。

你拿到的字符串看起来类似：

```
userId=123456789; serviceToken=AAAA....; xiaomichatbot_ph=BBBB....; ...
```

**必须包含这三个字段才能用**：`userId`、`serviceToken`、`xiaomichatbot_ph`。

### 9.3 推荐方式：通过 WebUI 添加（最简单）

项目内置了凭证导入功能，**不用你自己手写 JSON**。

1. 先按第九步把服务启动起来。
2. 浏览器打开 http://127.0.0.1:8000/
3. 找到"用户管理"区域，点"添加用户"。
4. 把上面复制的 Cookie 字符串整段粘贴进输入框，提交。
5. 系统会自动解析并在 `users/` 目录下生成 `user_你的userId.json` 文件。

成功添加后，控制面板能看到这个用户的实例状态（INIT / CREATING / AVAILABLE 等）。

### 9.4 手动方式：直接创建 JSON 文件

如果不想用 WebUI，可以手动在 `users/` 目录建文件，命名格式 `user_你的userId.json`，内容：

```json
{
  "userId": "你的userId数字",
  "serviceToken": "粘贴 serviceToken 的完整值",
  "xiaomichatbot_ph": "粘贴 xiaomichatbot_ph 的完整值",
  "name": "随便起个名字方便辨认"
}
```

例如：

```bash
nano users/user_123456789.json
```

粘贴上面格式的内容，把字段值换成你自己 Cookie 里对应的值，保存退出。

### 9.5 没有凭证怎么办

**没有凭证服务也能起来**，只是没有可用节点，调用 API 会返回 `Gateway Error: 没有可用的内网节点`。可以先空着，跑通启动流程后再补。

### 9.6 多账号轮询

可以同时放多个 `user_xxx.json` 文件，网关会自动在多个账号之间负载均衡，账号越多越稳定。

### 9.7 凭证什么时候会失效

- 小米账号在别处重新登录会导致 `serviceToken` 失效。
- 长时间不用也可能过期。
- 失效后该节点会进入 15 分钟冷却（401 冷却），日志里能看到提示。
- 凭证过期后需要重新走 9.2 步导出新 Cookie 替换。

### 9.8 确认放置成功

```bash
ls users/
```

应该能看到 `user_xxx.json` 文件。

## 12. 第十一步：测试 API 接口

### 12.1 用 curl 测试模型列表

新开一个终端窗口（不要关掉跑服务的那个），执行：

```bash
curl http://127.0.0.1:8000/v1/models
```

如果你设置了 `MIMO_RELAY_OPENAI_KEY`，要带上 Authorization 头：

```bash
curl http://127.0.0.1:8000/v1/models \
  -H "Authorization: Bearer 你设置的key"
```

应该返回一段 JSON，包含 `mimo-v2.5-pro`、`mimo-v2.5` 等模型列表。

### 12.2 测试 Chat Completions

```bash
curl http://127.0.0.1:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你设置的key" \
  -d '{
    "model": "mimo-v2.5",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": false
  }'
```

如果有可用节点，会返回模型生成的回复 JSON。如果没节点，会返回 `Gateway Error: 没有可用的内网节点`，先回到第八步把凭证补上。

### 12.3 在客户端里配置

任何支持 OpenAI 协议的客户端（ChatGPT Next Web、Cherry Studio、LobeChat、Cline 等）都可以连：

- **Base URL / API 端点**：`http://127.0.0.1:8000/v1`
- **API Key**：你在 `.env` 里设置的 `MIMO_RELAY_OPENAI_KEY`
- **模型名**：从 `/v1/models` 返回的列表里选

## 13. 第十二步：让服务在后台一直跑

前台运行的话，关闭终端服务就停了。要让它常驻，有几种方案，按难度从低到高介绍。

### 13.1 方案 A：nohup（最简单）

适合临时挂起，不太适合生产环境，但够用。

先 Ctrl+C 停掉前台运行的服务，然后执行：

```bash
nohup python main.py > app.log 2>&1 &
```

解释：

- `nohup`：让进程忽略挂断信号，关闭终端也不会停。
- `> app.log 2>&1`：把输出和错误都写到 `app.log` 文件。
- `&`：放后台运行。

执行后会输出一个进程 ID 数字，记一下。

**查看日志**：

```bash
tail -f app.log
```

按 `Ctrl + C` 退出查看（不会停服务）。

**停止服务**：

```bash
# 方法 1：用进程 ID（启动时记下来的那个）
kill 12345

# 方法 2：找出来再杀
ps aux | grep "python main.py"
kill 找到的进程ID
```

### 13.2 方案 B：screen（推荐入门）

`screen` 让你随时可以"回到"运行服务的终端。

安装（macOS 自带，Linux 可能要装）：

```bash
sudo apt install -y screen   # Ubuntu / Debian
sudo yum install -y screen   # CentOS
```

启动一个名为 `mimo` 的会话：

```bash
screen -S mimo
```

进入会话后，照常激活环境、启动服务：

```bash
source .venv/bin/activate
python main.py
```

按 `Ctrl + A` 然后按 `D`，会"detach"（离开但不关闭）会话，回到原终端，服务继续运行。

**重新进入会话查看**：

```bash
screen -r mimo
```

**停止服务**：进入会话按 `Ctrl + C`，再 `exit` 关闭 screen。

### 13.3 方案 C：systemd（推荐生产 / Linux 服务器）

让系统开机自动启动、崩溃自动重启。仅 Linux 适用。

#### 13.3.1 创建服务文件

```bash
sudo nano /etc/systemd/system/mimo2api.service
```

粘贴下面的内容（**注意修改路径和用户名**）：

```ini
[Unit]
Description=mimo2api Gateway Service
After=network.target

[Service]
Type=simple
User=你的Linux用户名
WorkingDirectory=/Users/admin/Desktop/本地部署/mimo2api/mimi3-master
Environment="PATH=/Users/admin/Desktop/本地部署/mimo2api/mimi3-master/.venv/bin"
ExecStart=/Users/admin/Desktop/本地部署/mimo2api/mimi3-master/.venv/bin/python main.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

把 `你的Linux用户名` 改成 `whoami` 命令查到的用户名，路径根据你服务器上的实际位置调整。

按 `Ctrl + O` 保存，`Ctrl + X` 退出。

#### 13.3.2 启用并启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable mimo2api
sudo systemctl start mimo2api
```

#### 13.3.3 管理命令

```bash
sudo systemctl status mimo2api    # 查看状态
sudo systemctl stop mimo2api      # 停止
sudo systemctl restart mimo2api   # 重启
sudo journalctl -u mimo2api -f    # 实时看日志
```

### 13.4 方案 D：launchd（macOS 后台开机自启）

macOS 上等价于 systemd 的方案。创建文件：

```bash
nano ~/Library/LaunchAgents/com.mimo2api.gateway.plist
```

内容：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.mimo2api.gateway</string>
  <key>WorkingDirectory</key>
  <string>/Users/admin/Desktop/本地部署/mimo2api/mimi3-master</string>
  <key>ProgramArguments</key>
  <array>
    <string>/Users/admin/Desktop/本地部署/mimo2api/mimi3-master/.venv/bin/python</string>
    <string>main.py</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/Users/admin/Desktop/本地部署/mimo2api/mimi3-master/logs/launchd.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/admin/Desktop/本地部署/mimo2api/mimi3-master/logs/launchd.err</string>
</dict>
</plist>
```

加载启动：

```bash
launchctl load ~/Library/LaunchAgents/com.mimo2api.gateway.plist
```

卸载停止：

```bash
launchctl unload ~/Library/LaunchAgents/com.mimo2api.gateway.plist
```

## 14. 第十三步：开放外部访问（可选）

### 14.1 防火墙开放 8000 端口

**Ubuntu / Debian**（用 ufw）：

```bash
sudo ufw allow 8000/tcp
sudo ufw status
```

**CentOS / RHEL**（用 firewalld）：

```bash
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

**macOS**：系统设置 → 网络 → 防火墙 → 允许 Python 接受连接。或直接关掉防火墙（不推荐）。

### 14.2 局域网访问

在同一 WiFi 下的其他设备访问：

```
http://你的服务器局域网IP:8000/
```

查找局域网 IP：

```bash
# macOS / Linux
ifconfig | grep "inet "
```

找形如 `192.168.x.x` 或 `10.0.x.x` 的地址。

### 14.3 公网访问（家用宽带需要内网穿透）

家庭宽带一般没有公网 IP，需要内网穿透工具：

- **frp**（开源、免费、要自己有 VPS）
- **cpolar / ngrok**（有免费额度，无需 VPS）
- **花生壳**（国内老牌）

具体配置参考各自官方文档。如果是云服务器（VPS）则跳过这一步，直接用公网 IP 即可。

### 14.4 强烈建议

公网暴露**必须**设置 `MIMO_RELAY_OPENAI_KEY` 和 `MIMO_WEBUI_PASSWORD`，否则会被人滥用甚至攻击。

最好再在前面加一层 Nginx 反代 + HTTPS（Let's Encrypt 免费证书）。

## 16. 常见问题与排错

### 16.1 启动报错 `ModuleNotFoundError: No module named 'xxx'`

依赖没装好。重新执行：

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### 16.2 启动报错 `ModuleNotFoundError: No module named 'fcntl'`

你在 Windows 上直接跑了。`fcntl` 是 Unix 专属，Windows 必须用 WSL2 才行。

### 16.3 启动报错 `当前进程锁被占用`

已经有一个实例在跑，或者上次没正常退出留下了锁文件。

```bash
# 查找运行中的进程
ps aux | grep "python main.py"

# 如果确认没在跑，删掉锁文件
rm mimo2api.lock
```

### 16.4 启动报错 `Address already in use` 或 `端口已被占用`

8000 端口被别的程序占了。

```bash
# 查谁占了 8000
# macOS / Linux
lsof -i :8000

# 杀掉占用进程
kill 占用的进程ID
```

或者改 `.env` 里的 `SERVER_PORT` 换个端口。

### 16.5 调用 API 返回 `Gateway Error: 没有可用的内网节点`

`users/` 目录下没有有效凭证文件，或者节点还没连上来。

- 检查 `ls users/` 是否有 `user_xxx.json`
- 检查凭证是否过期（账号失效会进入 401 冷却 15 分钟）
- 看日志 `tail -f logs/gateway.log` 找原因

### 16.6 调用 API 返回 `401 Unauthorized`

客户端没带 Authorization 头，或者 Key 和 `.env` 里的对不上。

### 16.7 WebUI 登录后立刻跳回登录页

- `.env` 里 `MIMO_WEBUI_SECRET` 没设置或太短，浏览器换个无痕窗口试试。
- 浏览器禁用了 Cookie，去设置里打开。

### 16.8 局域网设备访问不了

- 确认 `.env` 里 `SERVER_HOST=0.0.0.0`（不是 127.0.0.1）。
- 确认防火墙开放了 8000 端口（看第 14.1 节）。
- 确认两台设备在同一个网段。

### 16.9 pip 安装巨慢或失败

换国内源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

或者豆瓣源：

```bash
pip install -r requirements.txt -i https://pypi.douban.com/simple
```

### 16.10 服务跑一段时间就崩溃

看 `logs/gateway.log` 找原因。配合 systemd 的 `Restart=on-failure` 可以自动拉起。
